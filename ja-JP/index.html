Architecture

Remote agent を長く動かすための設計。

Azents は session loop、Runtime boundary、provider ごとの実行を分け、実インフラ上でも agent の作業が続くようにします。

01 / session loop

Runtime execution から decouple された独立 session loop。独立した session loop。

Engine が agent loop と durable events を扱い、Runtime は files、tools、shell、compute が必要なときだけ呼び出されます。

01 / 03

02 / runtime adapter

Provider はひとつの Runtime adapter の後ろに置きます。ひとつの adapter、複数の provider。

まずは Kubernetes からです。同じ adapter 形状で local machine、E2B、custom backend へ広げられるようにします。

02 / 03

03 / long-running agent

Long-running agent には durable state が必要です。長い run に durable state を。

Tab が閉じても、worker が移っても、実行環境が再作成されても、history、progress、handoff point は残るべきです。
