# Database Migration (azents)

Create and manage Alembic database migrations for azents.

- **App directory**: `python/apps/azents`
- **Alembic directory**: `python/apps/azents/db-schemas/rdb`
- **Models**: `python/apps/azents/src/azents/rdb/models/`
- **Revision file**: `python/apps/azents/db-schemas/rdb/revision`
- **Docker Compose**: `docker-compose.azents.yaml`

## Workflow

1. **Load environment variables and navigate**:
   ```bash
   cd /path/to/azents/python/apps/azents && set -a && source .env && set +a && cd db-schemas/rdb
   ```
   > azents must load database connection information from the `.env` file.

2. **Generate migration template**:
   ```bash
   uv run alembic revision -m "description"
   ```

3. **Edit the generated migration file** in `migrations/versions/`:
   - Fill in `upgrade()` and `downgrade()` functions
   - Remove any auto-generated comments if using `--autogenerate`

4. **Update revision file**:
   - Update `db-schemas/rdb/revision` with the new migration's revision ID

## Auto-generation (Optional)

For schema changes that follow SQLAlchemy model changes:
```bash
uv run alembic revision --autogenerate -m "description"
```

**Important**: Always review autogenerated migrations - they may not catch all changes correctly.

## Best Practices

- Keep migrations atomic and focused on a single logical change
- Use descriptive migration messages
- Never manually create migration files without using `alembic revision`
- Test both upgrade and downgrade paths when possible
