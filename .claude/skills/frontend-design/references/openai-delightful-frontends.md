# OpenAI Delightful Frontends Reference

Source: [Designing delightful frontends with GPT-5.4](https://developers.openai.com/blog/designing-delightful-frontends-with-gpt-5-4)

## Principles For Product Tools

- Default to operational clarity for dashboards, admin tools, and workflow surfaces.
- Prioritize orientation, status, and action over promise, mood, or brand voice.
- Start with the working surface itself: filters, tables, task context, media, status, or other primary controls.
- Use headings that describe what the area is or what the user can do there.
- Use concise supporting text only when it explains scope, behavior, freshness, or decision value.
- If a section does not help someone operate, monitor, compare, or decide, remove it.

## Visual Direction

- Use a strong real visual anchor when the domain has real objects or media.
- Avoid decorative texture as the only first-viewport signal.
- Keep the interface restrained: few colors, clear spacing, readable density, and minimal chrome.
- Use motion only when it improves hierarchy or affordance.
- Avoid generic SaaS card grids, filler copy, ornamental gradients, and app UIs made of stacked cards.

## Practical Litmus Checks

- Is the product or task unmistakable in the first screen?
- Is there one dominant idea in each section?
- Can the user understand the workflow by scanning labels and numbers only?
- Are cards necessary, or would a pane/table/toolbar be clearer?
- Would the design still feel polished if decorative shadows were removed?
