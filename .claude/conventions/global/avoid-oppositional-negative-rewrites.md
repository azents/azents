# Avoid oppositional negative rewrites

- AVOID turning corrective feedback into defensive text that says both the corrected statement and an unnecessary negation of the rejected statement.
- Prefer the direct corrected statement.
- Exception: mention the rejected alternative only when it materially clarifies the context or prevents a likely misunderstanding.

## Bad

```markdown
Use `azcommon.types.JSONObject` for metadata rather than adding a duplicate JSON alias.
```

## Good

```markdown
Use `azcommon.types.JSONObject` for metadata.
```
