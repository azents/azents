# Alembic Migrations: Always Use `alembic revision`

Hand-written migration files miss the boilerplate Alembic needs (file naming, `down_revision` chain, `branch_labels`) and silently break the migration graph.

- ALWAYS create migrations with `alembic revision -m "<message>"` or `alembic revision --autogenerate -m "..."`
- Even for the simplest one-line schema change, generate the file first, then edit
- ALWAYS review autogenerated content before committing — autogen misses ENUM rename, server defaults, and constraint renames
- Remove the auto-generated header comments
- After generation, update `db-schemas/rdb/revision` with the new revision ID

## Bad

```bash
# Hand-creating a migration file
touch db-schemas/rdb/versions/20260501_add_column.py
# Type out upgrade/downgrade by hand
```

## Good

```bash
cd python/apps/<app>
uv run alembic revision --autogenerate -m "add foo column to bar"
# Review and edit the generated file, then update db-schemas/rdb/revision
```
